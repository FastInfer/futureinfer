# Research Notes

Working notes that ground FutureInfer's open questions in primary sources. Each file is a living document — start with what we know, where it came from, and where to dig next.

## Files

- [frameworks.md](frameworks.md) — Open-source agent frameworks: who is in the space, what each does, where they fall short.
- [failure-modes.md](failure-modes.md) — Academic taxonomies of how and why LLM agents fail.
- [industry-reports.md](industry-reports.md) — Primary practitioner accounts from labs running agents in production.
- [incidents.md](incidents.md) — Concrete production failures, CVEs, and security incidents.
- [standards.md](standards.md) — Emerging security and governance standards for agentic systems.
- [sandboxing.md](sandboxing.md) — The execution-isolation layer below the framework.

## Method

Prefer primary sources: arxiv papers, CVE entries, lab engineering blogs, OWASP. Secondary sources (marketing blogs, listicles) are not load-bearing for any claim — if a claim depends on one, mark it.

Each file ends with a **Threads to go deeper** section. That is the to-do list for the next research pass.
